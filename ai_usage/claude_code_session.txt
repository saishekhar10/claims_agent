     Plan: Image Transcription Cache

     Context

     transcribe_image() and assess_image_handwriting_quality() are called every
     time preprocess_claim_folder() runs — including every time the user
     provides new input in the interactive loop, which re-preprocesses the
     original claim folder. These are the most expensive operations:
     transcribe_image is a full LLM vision call per image. Caching to disk means
      each image is only processed once per claim, across all runs and sessions.

     Plan: Codebase Redundancy Cleanup

     Context

     Audit identified redundancies across src/main.jac and src/tools.jac. Most
     are a structural consequence of Jac's ::py:: blocks — each block is an
     isolated Python scope, so import json, import os, and from rich.console 
     import Console must be repeated per block. These cannot be eliminated
     without restructuring. However, four genuine redundancies can be fixed
     cleanly.

     ---
     Fix 1 — Extract repeated process→save→display pattern (src/main.jac)

     The sequence below appears 4 times in run_interactive_loop (choices 1,
     2-text, 2-image, 3), only differing in the result variable name:

     new_result_N: ClaimResult = ClaimResult();
     ::py::
     with Console().status(f"Agent is processing {claim_id}...",
     spinner="dots"): pass
     ::py::
     new_result_N = process_claim(claim_id, doc_contents_json);
     current_result_json = serialize_result(new_result_N);
     save_result(claim_id, current_result_json);
     save_docs(claim_id, doc_contents_json);
     display_result_dict(claim_id, current_result_json);

     Fix: Add a Jac helper function run_agent(claim_id: str, doc_contents_json: 
     str) -> str that runs this sequence and returns the updated result_json.
     Place it just before run_interactive_loop. Each of the 4 call sites
     becomes:

     current_result_json = run_agent(claim_id, doc_contents_json);

     The spinner lives inside run_agent. The new_result_N temp variables
     disappear entirely. current_result_json is updated by the return value.

     Fix 2 — Merge two consecutive ::py:: blocks that split for no reason 
     (src/main.jac)
